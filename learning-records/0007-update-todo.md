# 第六关学习记录：更新 Todo

- 日期：2026-07-20
- 状态：已通关

## 我观察到了什么

- `PUT /todos/1` 返回 `200 OK` 和更新后的 Todo。
- 随后 `GET /todos/1` 返回相同的新标题和完成状态，证明内存数据已经改变。
- 请求中的 `flase` 不是合法 JSON 布尔值，因此返回 `400` 和 `请求 JSON 无效`。
- 改成合法的 `false` 后，空标题返回 `400` 和 `title 不能为空`。

## 我的理解

- 路径 ID 标识本次要更新哪一条数据。
- 请求 JSON 提供准备写入的新数据。
- `todos[index]` 定位列表中匹配的元素，赋值操作会用新 Todo 替换旧数据。

## 术语校准

- `flase` 触发的是 JSON 解析错误，代码还没有进入标题校验。
- `false` 让 JSON 正常解码，随后空标题才会触发业务校验。
- 同样是 `400 Bad Request`，不同响应正文可以帮助定位失败发生在哪一层。

## 当前掌握证据

- 能使用 `PUT` 更新 Todo，并用后续 `GET` 验证内存状态。
- 能解释路径 ID、请求 JSON 和 slice 位置如何共同完成更新。
- 能通过修正布尔值拼写，区分解析失败与业务规则失败。

## 下一次复现

第七关将用 `DELETE /todos/{id}` 删除 Todo，并用 `204 No Content` 与后续 `404 Not Found` 证明删除成功。
