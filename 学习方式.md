# Go 学习方式

> 新的日常入口是 [CURRENT.md](CURRENT.md)。学习时只打开当前关；完整首月关卡放在教师用的 [COURSE-DESIGN.md](COURSE-DESIGN.md)，不需要提前阅读。

## 学习目标

从前端逐步承担后端需求，优先建立阅读、运行、修改和排错 Go 后端代码的能力。

## 学习契约

- 每次只学一个足够小的目标，最好在 1–2 小时内完成。
- 起步阶段继续缩小目标，单个台阶控制在约 10–20 分钟。
- 每次先明确本次会用到的 Go 概念，再进行讲解，然后拆解做法，最后实现。
- 采用“AI 主写、自己主理解和验证”的方式，不要求逐行手敲所有代码。
- AI 可以生成样板代码和大部分代码，但自己需要做到：
  - 读懂每个代码块的作用；
  - 能运行并验证结果；
  - 至少亲自完成一个小改动；
  - 能解释报错的大致原因，并参与排错。
- 通过一个问题一个问题地确认理解，避免一次引入过多概念。

## 第一阶段边界

- 使用 Go 标准库，不引入 Gin 等框架。
- 暂不使用数据库，数据先保存在内存中。
- 先建立“代码 → 启动 → 请求 → 响应”的完整反馈回路。

## 第一课拆分

1. 启动服务并实现 `GET /hello`，返回 `Hello, Go`。
2. 返回一个 Todo 的 JSON，认识 `struct` 和 JSON。
3. 返回多个 Todo，认识切片 `[]Todo`。

## 第一课当前任务

实现一个最小 Go HTTP 服务：访问 `GET /hello` 时返回：

```text
Hello, Go
```
