# 第一个月教师用关卡设计

> 这是课程后台，不是每日待办。学习时只打开 `CURRENT.md`。

## 设计判断

参考仓库是一门“Node.js/SQL 后端工程师迁移到 Go”的 36 天工业课程，不是零基础 Go 课。它的单课通常同时要求理解 4–8 个概念、编写多个文件并完成测试，适合有后端心智模型的人，不适合纯新手直接照走。

本月只保留对“接手小型 Go 后端改动”收益最高的闭环：HTTP 请求与响应、函数、struct、JSON、slice、输入校验、状态码、最小测试和基本排错。其余主题等真实任务需要时再解锁。

## 通关协议

每个微关只包含四步：

1. **回忆**：回答一个上关问题，最多 1 分钟。
2. **行动**：运行或修改一个很小的代码点，约 5–12 分钟。
3. **验证**：用命令、浏览器或测试立刻看到结果。
4. **复述**：用一句自己的话解释刚才发生了什么。

标准关 10–20 分钟。Boss 关 20–30 分钟，只组合旧能力。没有“落后”；中断后从未完成的当前关继续。

## 首月关卡池

课程按能力自适应解锁，不按日历强制推进。目标是一个月完成 12–16 个核心微关；少于 12 个也不补课。

### 世界一：让服务说话

1. 启动 `GET /hello`，看见 `Hello, Go`
2. 改一段响应文字，重新验证
3. 新增 `GET /ping`，返回 `pong`
4. Boss：画出“请求 → handler → 响应”三步链路

### 世界二：让服务返回数据

5. 把响应文字放进变量和函数
6. 用 `struct` 表示一个 Todo
7. 把一个 Todo 编码成 JSON
8. 用 `[]Todo` 返回多个 Todo

### 世界三：让服务理解输入

9. 从 query string 读取名字
10. 从路径读取 Todo ID
11. 输入缺失时返回 `400`
12. Boss：完成“按 ID 查 Todo”的最小接口

### 世界四：安全地修改代码

13. 读懂并修复一个编译错误
14. 为成功响应写第一个 handler 测试
15. 为错误响应补一个测试
16. Boss：给 Todo 增加 `done` 字段并保持测试通过

## 延后清单

- **第二个月候选**：显式 `error`、package 拆分、小接口、context、middleware、更多测试。
- **真实需求出现再学**：数据库、migration、事务、鉴权、部署。
- **明确不在首月**：gRPC、protobuf、goroutine、channel、race detector、Agent 抽象。

## 动态规则

- 如果某关连续两次需要大量提示，插入一关“同构练习”，不推进新概念。
- 如果学习者能运行、修改并解释，则下一关只增加一个新变量。
- 每次只更新 `CURRENT.md` 指向下一课；不要把完整关卡池复制到当前页。
