写入与查询

上一关只确认 MySQL 和目标数据库存在。这一关创建第一张表，把一条 Todo 真正写进数据库，再把它查询出来。

本关胜利画面

todos 表创建成功，插入的自定义 Todo 能被 SELECT 查询出来；重启 MySQL 容器后，这条数据仍然存在。

1. 进入目标数据库

先启动 MySQL，并确认它已经健康运行：

docker compose up -d
docker compose ps

进入客户端：

docker compose exec mysql \
  mysql -ugo_learning -pgo_learning go_learning

看到 mysql> 后，先确认当前数据库：

SELECT DATABASE();

必须返回 go_learning。这一步是在确认后面的表会建在哪里。

2. 创建 todos 表

执行下面的 SQL：

CREATE TABLE IF NOT EXISTS todos (
  id INT PRIMARY KEY AUTO_INCREMENT,
  title VARCHAR(255) NOT NULL,
  done BOOLEAN NOT NULL DEFAULT FALSE
);

IF NOT EXISTS 表示表已经存在时不要报错。三个字段先这样理解：

  • id 是自动递增的主键，每行都有自己的编号。
  • title 保存 Todo 文字，而且不能留空。
  • done 保存完成状态，默认是 false。

3. 查看表结构

DESCRIBE todos;

从结果中找到 Field、Type、Null、Key 和 Default。重点指出：哪个字段是主键，哪个字段不能为 NULL，哪个字段有默认值。

4. 写入自己的 Todo

把标题改成你自己的文字，再执行：

INSERT INTO todos (title)
VALUES ('学习 SQL');

这里只提供 title，因为 id 会自动生成，done 会使用默认值 false。

5. 查询刚写入的数据

SELECT id, title, done
FROM todos
ORDER BY id;

结果中应该能看到你刚才写入的标题。再加一个条件，只查询未完成的 Todo：

SELECT id, title, done
FROM todos
WHERE done = FALSE
ORDER BY id;

把 WHERE 条件或标题改一处，观察查询结果怎样变化。INSERT 负责写入一行，SELECT 负责读取符合条件的行。

6. 证明数据没有回到内存

退出客户端，停止并重新启动容器：

exit
docker compose stop
docker compose up -d

再次进入客户端，执行上一节的 SELECT。如果 Todo 仍然存在，说明数据由 Docker volume 保存，不会因为容器停止而消失。
