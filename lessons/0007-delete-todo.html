删除 Todo Boss

完成最后一个内存操作：删除指定 Todo。随后把创建、查询、更新、删除连成一套完整 CRUD。

本关胜利画面

DELETE /todos/3 返回 204 No Content；随后查询 ID 3 返回 404 Not Found。

1. 先创建一条练习数据

启动服务后，创建一条专门用于删除的 Todo：

curl -i -X POST http://localhost:8080/todos \
  -H 'Content-Type: application/json' \
  -d '{"title":"准备删除的 Todo","done":false}'

记下响应中的 ID。服务刚重启时它应是 3；后续删除请求要使用你实际看到的 ID。

2. 删除它

curl -i -X DELETE http://localhost:8080/todos/3

成功响应只有状态，没有 JSON 正文：

HTTP/1.1 204 No Content

204 No Content 表示操作成功，但服务器不返回响应正文。不要为它额外写 JSON。

3. 证明它真的消失了

curl -i http://localhost:8080/todos/3

此时应看到 404 Not Found。一次 204 说明删除请求成功，后续 404 则从查询角度证明资源已经不存在。

4. 看懂 slice 删除

todos = append(todos[:index], todos[index+1:]...)

todos[:index] 是目标之前的部分，todos[index+1:] 是目标之后的部分。把前后两段重新接起来，中间的目标元素就被移除了。

创建 Todo 现在使用单独递增的 nextTodoID，不会因为列表变短而重复使用已有 ID。

5. CRUD Boss 复述

POST   /todos       → 创建 → 201 Created
GET    /todos/{id}  → 查询 → 200 OK / 404 Not Found
PUT    /todos/{id}  → 更新 → 200 OK / 400 Bad Request / 404 Not Found
DELETE /todos/{id}  → 删除 → 204 No Content / 404 Not Found

不用背代码，只要能从“客户端想做什么”选择方法和路径，再根据结果理解状态码，就已经完成了第一轮 REST CRUD。
