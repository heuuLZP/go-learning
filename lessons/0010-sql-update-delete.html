完成 SQL CRUD

上一关已经会写入和查询。本关先用两个字段创建练习数据，再用 UPDATE 和 DELETE 精确操作一行，最后自己完成一轮 SQL CRUD。

本关胜利画面

一条 Todo 从写入、查询、更新到删除都由你亲自完成；删除后的查询返回 Empty set，而其他 Todo 不受影响。

1. 把长 SQL 拆成四个部分

SQL 不需要一次背完整句子。先判断动作，再依次补上表、内容和条件：

创建：INSERT INTO 表 (字段) VALUES (值)
读取：SELECT 字段 FROM 表 WHERE 条件
更新：UPDATE 表 SET 新值 WHERE 条件
删除：DELETE FROM 表 WHERE 条件

SQL CRUD 中的 Create 对应 INSERT，Read 对应 SELECT，Update 和 Delete 与英文名称一致。

2. 同时写入两个字段

启动 MySQL，并进入 go_learning 数据库：

docker compose up -d
docker compose exec mysql \
  mysql -ugo_learning -pgo_learning go_learning

看到 mysql> 后执行：

INSERT INTO todos (title, done)
VALUES ('第十关练习', FALSE);

字段和值按位置一一对应：title 接收文字，done 接收 FALSE。id 仍由数据库自动生成。

查询全部数据，记住这条练习 Todo 的实际 ID：

SELECT id, title, done
FROM todos
ORDER BY id;

3. 先用 WHERE 锁定目标

下面示例假设练习 Todo 的 ID 是 2。如果你看到的 ID 不同，后续所有语句都要换成你的实际 ID。

SELECT id, title, done
FROM todos
WHERE id = 2;

WHERE 决定哪些行会被操作。在执行 UPDATE 或 DELETE 前，先用相同条件执行一次 SELECT，确认它只找到目标行。

4. 更新这一行

UPDATE todos
SET title = '已完成第十关更新', done = TRUE
WHERE id = 2;

SET 后面写新值，多个字段之间使用逗号分隔；WHERE 指定要修改的行。成功后再次查询：

SELECT id, title, done
FROM todos
WHERE id = 2;

应该看到新标题，并且 done 从 0 变成 1。

5. 删除这一行

先重复上一节的 SELECT，确认目标；然后执行：

DELETE FROM todos
WHERE id = 2;

用相同条件再次查询：

SELECT id, title, done
FROM todos
WHERE id = 2;

结果应为 Empty set。最后查询全表，确认其他 Todo 仍然存在。

安全规则

本关不要执行没有 WHERE 的 UPDATE 或 DELETE：前者会更新所有行，后者会删除所有行。先 SELECT 同一个条件，是数据库操作中很实用的安全习惯。

6. SQL CRUD Boss

暂时遮住上面的完整语句，只看下面四个动作，创建另一条自定义 Todo：

  1. 同时写入自定义 title 和 done。
  2. 根据实际 ID 查询它。
  3. 把标题和完成状态更新一次，再查询验证。
  4. 删除它，再用查询证明它已经不存在。

卡住时只回看第一节的语句骨架，再自己补表名、字段和值。目标是从动作推回 SQL 结构，不是一次就把所有标点背熟。
