更新 Todo

这一关组合两条已经走过的数据路线：从 URL 读取 ID，从请求正文读取 JSON，然后替换内存列表中的 Todo。

本关胜利画面

PUT /todos/1 返回 200 OK 和更新后的 Todo；随后再次查询 ID 1，看到新标题和新的完成状态。

1. 先看 PUT 的输入

路径：/todos/1
正文：{"title":"完成第六关","done":true}

路径 ID 决定更新哪个 Todo，JSON 正文提供它的新内容。这里使用 PUT 表示完整更新，所以同时发送 title 和 done。

2. 发出更新请求

启动服务后，在另一个终端运行：

curl -i -X PUT http://localhost:8080/todos/1 \
  -H 'Content-Type: application/json' \
  -d '{"title":"完成第六关","done":true}'

找到响应状态和更新结果：

HTTP/1.1 200 OK
Content-Type: application/json

{"id":1,"title":"完成第六关","done":true}

ID 来自 URL，服务端会覆盖正文中可能出现的 ID，确保更新的仍是路径指定的 Todo。

3. 复用两段旧知识

id, err := strconv.Atoi(r.PathValue("id"))
json.NewDecoder(r.Body).Decode(&updatedTodo)

第一行复用第五关，从路径取得数字 ID；第二行复用第四关，从请求正文得到新的 Todo 内容。

4. 替换列表中的元素

for index := range todos {
    if todos[index].ID == id {
        updatedTodo.ID = id
        todos[index] = updatedTodo
        return
    }
}

index 是元素在 slice 中的位置。找到相同 ID 后，用 todos[index] = updatedTodo 替换该位置的旧数据。

5. 验证更新与错误

  1. 运行 curl -i http://localhost:8080/todos/1，确认更新结果仍然存在。
  2. 把标题改成你自己的文字，再发送一次 PUT。
  3. 发送标题为空的 {"title":"","done":false}，确认返回 400 Bad Request。

第四关的 {} 可以创建空标题 Todo；这一关增加了业务校验，空标题即使是合法 JSON，也会返回 400。暂时只校验标题；严格检查 done 是否出现将在以后需要时再做。

6. 一题验收

todos[index] = updatedTodo 做了什么？
