Go 连接 MySQL

上一关是在 MySQL 客户端里完成 CRUD。这一关让 Go 服务接管“查询列表”和“创建 Todo”：请求进入 handler 后，通过 database/sql 和 MySQL 驱动读写同一张 todos 表。

本关胜利画面

启动 Go 服务后，GET /todos 能读到 MySQL 中已有的 Todo；用 POST /todos 创建一条数据，再直接进入 MySQL 查询也能看到它。这两个动作不再依赖那份内存切片。

1. 先画出新的请求链路

HTTP 请求
  ↓
todosHandler
  ↓
database/sql → MySQL 驱动 → MySQL
  ↓
rows.Scan → Todo → JSON 响应

database/sql 是 Go 提供的数据库操作接口；真正认识 MySQL 协议的是驱动。本关使用 go-sql-driver/mysql，但查询仍然通过标准库完成。

2. 启动 MySQL 并确认数据还在

docker compose up -d

先确认第十关留下的表和数据：

docker compose exec mysql \
  mysql -ugo_learning -pgo_learning go_learning \
  -e "SELECT id, title, done FROM todos ORDER BY id;"

3. 让 Go 先完成一次 Ping

在 main.go 中加入数据库包和驱动：

import (
  "database/sql"
  _ "github.com/go-sql-driver/mysql"
)

const dsn = "go_learning:go_learning@tcp(127.0.0.1:3306)/go_learning?parseTime=true"

func openDB() (*sql.DB, error) {
  db, err := sql.Open("mysql", dsn)
  if err != nil {
    return nil, err
  }
  if err := db.Ping(); err != nil {
    db.Close()
    return nil, err
  }
  return db, nil
}

不要只单独添加 import 就保存：很多编辑器会用 goimports 自动整理 import，而此时 database/sql 还没有被 openDB 使用，会被判定为未使用并移除。请把上面的 import、dsn 和 openDB 一起粘贴，再保存 main.go。

保存完整代码后再安装依赖：

go get github.com/go-sql-driver/mysql
go mod tidy

go get 会把驱动写入 go.mod 和 go.sum。必须先保存驱动导入，再执行 go mod tidy；否则 tidy 会把尚未被源码使用的依赖移除。

这里的下划线导入不是多余的：它只执行驱动的注册代码。sql.Open 创建的是连接池句柄，不保证 MySQL 此刻可达；Ping 才是本关要观察的连通性检查。

在 main 中启动 HTTP 服务前调用：

db, err := openDB()
if err != nil {
  log.Fatal("连接 MySQL 失败：", err)
}
defer db.Close()

运行 go run .。如果密码、端口或容器状态不对，先读错误信息；不要先改 SQL。

4. 把 GET /todos 改成查询数据库

把原来的内存列表 handler 改成接收 *sql.DB。先查询，再逐行扫描：

func todosHandler(db *sql.DB) http.HandlerFunc {
  return func(w http.ResponseWriter, r *http.Request) {
    rows, err := db.QueryContext(r.Context(),
      "SELECT id, title, done FROM todos ORDER BY id")
    if err != nil {
      http.Error(w, "查询 Todo 失败", http.StatusInternalServerError)
      return
    }
    defer rows.Close()

    result := make([]Todo, 0)
    for rows.Next() {
      var todo Todo
      if err := rows.Scan(&todo.ID, &todo.Title, &todo.Done); err != nil {
        http.Error(w, "读取 Todo 失败", http.StatusInternalServerError)
        return
      }
      result = append(result, todo)
    }
    if err := rows.Err(); err != nil {
      http.Error(w, "读取 Todo 失败", http.StatusInternalServerError)
      return
    }

    w.Header().Set("Content-Type", "application/json")
    json.NewEncoder(w).Encode(result)
  }
}

把路由注册改为 http.HandleFunc("GET /todos", todosHandler(db))。QueryContext 使用请求的取消信号，Scan 把 SQL 行的列放回 Go struct；rows.Close 和 rows.Err 是查询完成后的两个检查点。

重新启动服务并请求：

curl -i http://localhost:8080/todos

响应中的数据应该来自 MySQL，而不是 main.go 里原来的示例切片。先在 MySQL 客户端新增或修改一行，再刷新这个接口，观察结果是否同步。

5. 让 POST /todos 写入数据库

把原来的整个创建函数替换成下面的版本。注意它和列表 handler 一样接收 db，返回一个真正的 http.HandlerFunc：

func createTodoHandler(db *sql.DB) http.HandlerFunc {
  return func(w http.ResponseWriter, r *http.Request) {
    var todo Todo
    if err := json.NewDecoder(r.Body).Decode(&todo); err != nil {
      http.Error(w, "请求 JSON 无效", http.StatusBadRequest)
      return
    }
    if strings.TrimSpace(todo.Title) == "" {
      http.Error(w, "title 不能为空", http.StatusBadRequest)
      return
    }

    result, err := db.ExecContext(r.Context(),
      "INSERT INTO todos (title, done) VALUES (?, ?)",
      todo.Title, todo.Done)
    if err != nil {
      http.Error(w, "创建 Todo 失败", http.StatusInternalServerError)
      return
    }

    id, err := result.LastInsertId()
    if err != nil {
      http.Error(w, "读取新 Todo ID 失败", http.StatusInternalServerError)
      return
    }
    todo.ID = int(id)

    w.Header().Set("Content-Type", "application/json")
    w.WriteHeader(http.StatusCreated)
    if err := json.NewEncoder(w).Encode(todo); err != nil {
      log.Printf("写入 JSON 响应失败: %v", err)
    }
  }
}

问号是参数占位符，不要用字符串拼接把标题直接塞进 SQL。这样既能正确处理引号，也为后续防注入习惯打基础。然后把路由注册改为 http.HandleFunc("POST /todos", createTodoHandler(db))，再验证：

curl -i -X POST http://localhost:8080/todos \
  -H 'Content-Type: application/json' \
  -d '{"title":"created by go","done":false}'

curl -i http://localhost:8080/todos

再用 MySQL 客户端执行 SELECT，确认同一个 ID 和标题已经落在数据库中。
