按 ID 查询 Todo

上一关把 JSON 放进请求正文；这一关把查询目标放进 URL 路径，根据 ID 返回一个 Todo，或者明确告诉客户端它不存在。

本关胜利画面

GET /todos/1 返回 200 OK 和单个 Todo；GET /todos/999 返回 404 Not Found。

1. 看懂路径中的 ID

http.HandleFunc("GET /todos/{id}", todoHandler)

{id} 表示这段路径是可变化的。请求 /todos/1 时它是 1，请求 /todos/2 时它是 2。

2. 查询一个存在的 Todo

启动服务后，在另一个终端运行：

curl -i http://localhost:8080/todos/1

找到状态码和单个 JSON 对象：

HTTP/1.1 200 OK
Content-Type: application/json

{"id":1,"title":"看懂 JSON 响应","done":false}

这里返回的是一个 Todo，所以正文使用花括号；GET /todos 返回列表，才使用方括号。

3. 把路径文字变成数字

id, err := strconv.Atoi(r.PathValue("id"))

r.PathValue("id") 取出路径中的文字，strconv.Atoi 再把它转换成 Go 的 int，以便和 Todo.ID 比较。

如果路径是 /todos/abc，abc 不能转换成数字，服务会返回 400 Bad Request。

4. 在列表中寻找

for _, todo := range todos {
    if todo.ID == id {
        json.NewEncoder(w).Encode(todo)
        return
    }
}

range 依次查看列表中的 Todo。找到相同 ID 后写入响应并 return，handler 到这里结束。

5. 亲自制造一次不存在

  1. 先把路径 ID 改为 2，确认返回另一个 Todo。
  2. 再运行 curl -i http://localhost:8080/todos/999。
  3. 找到 404 Not Found 和正文 Todo 不存在。

404 表示请求格式没有问题，但服务器找不到这个 ID 对应的资源。它和“路径 ID 不是数字”的 400 含义不同。

6. 一题验收

GET /todos/999 为什么返回 404 而不是 400？
