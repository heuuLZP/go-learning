启动 MySQL

前七关的数据只活在 Go 内存中。现在先启动一个独立的 MySQL 服务，进入它的客户端，并确认目标数据库已经存在。

本关胜利画面

docker compose ps 显示 MySQL 为 healthy，进入客户端后执行 SELECT DATABASE() 得到 go_learning。

1. 认识三个边界

Docker Compose → 启动和配置服务
MySQL 容器    → 正在运行的数据库服务
go_learning   → MySQL 服务中的一个数据库

容器不是数据库本身的名字；一个 MySQL 服务中可以有多个数据库。这一关只确认这三层能连通。

2. 启动 MySQL

先确认 Docker Desktop 已经运行，然后在仓库根目录执行：

docker compose up -d

第一次会下载 MySQL 8.4 镜像，可能需要几分钟。随后查看状态：

docker compose ps

初始化期间可能先显示 health: starting；稍等片刻再次执行，直到看到 healthy。

3. 进入 MySQL 客户端

docker compose exec mysql \
  mysql -ugo_learning -pgo_learning go_learning

命令前半段进入名为 mysql 的 Compose 服务，后半段启动 MySQL 客户端，并连接到 go_learning 数据库。这些账号只用于本地学习环境。

4. 确认当前数据库

看到 mysql> 提示符后，逐条输入：

SHOW DATABASES;
SELECT DATABASE();

第一条应该列出 go_learning；第二条应该直接返回当前数据库名。SQL 语句以分号结束。

再亲自输入一条只返回文字的查询：

SELECT '我进入 MySQL 了' AS message;

5. 退出与停止

exit

退出客户端不会停止 MySQL 容器。需要暂停服务时运行：

docker compose stop

再次开始学习时运行 docker compose up -d。暂时不要运行 docker compose down -v，其中 -v 会删除保存数据库数据的 volume。

6. 一题验收

go_learning 在这套环境中是什么？

向老师通关

贴出 docker compose ps 的状态、SELECT DATABASE() 和自定义文字查询的结果，再用自己的话区分 Compose、MySQL 容器与 go_learning 数据库。
