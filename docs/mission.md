# Mission: 从前端走向 Go 后端

## Why
从前端逐步承担真实的后端需求，建立 HTTP、REST、SQL 和数据持久化的核心心智，并用 Go 完成一条可运行的后端链路，而不是只背语言语法。

## Success looks like
- 能说明 HTTP 请求如何经过 handler、业务代码和 MySQL，再变成 JSON 响应
- 能用 REST API 和 SQL 分别完成 Todo 的增删改查
- 能运行、修改和排错一个小型 Go 后端服务
- 能在 AI 生成样板代码后验证结果、亲自修改，并解释关键代码块

## Constraints
- 当前是 Go 纯新手，默认不依赖后端开发经验
- 每次只出现一个 10–20 分钟的小目标
- 按每周三次设计，首月约 12 个微关；完成 8 或 10 关也算有效通关
- 采用“AI 主写、自己主理解和验证”，不要求手敲样板代码
- 用运行结果获得即时反馈；连续学习不以补课或追赶进度为前提

## Out of scope
- 第一个月不学 Gin、ORM、Prisma、Redis、鉴权、事务和索引调优
- gRPC、并发和复杂项目结构等真实需求出现后再学
- 不追求一次学完整语法，也不以看完长教程作为完成标准
