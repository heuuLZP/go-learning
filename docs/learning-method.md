# 螺旋式微学习方法

> 这是一套可复用于编程、数据库、数学、语言或其他实践型主题的学习与教学方法。具体项目负责定义“学什么”，本文负责约束“怎样逐步学会”。

## 核心目标

学习路线应当同时做到：

- **先能行动，再逐步理解**：第一次只给完成当前动作所需的最小模型。
- **反复遇见，但不是原地重复**：同一概念在新场景中再次出现，每次增加一层深度。
- **主动提取，而不是只重读**：先回忆、预测或解释，再查看答案和运行结果。
- **主线稳定，局部自适应**：最终能力和阶段成果固定，关内练习根据实际疑问调整。

## 双层导航与闯关机制

一条学习路线同时需要全局方向和眼前动作，但两者不应堆在同一个页面里。

### 大纲式路线图

路线图是一份稳定、简洁的全局地图，只展示：

- 最终要获得的可观察能力；
- 由真实成果划分的少量阶段；
- 每个阶段包含的关卡标题和当前状态；
- 当前所处位置以及整体进度。

路线图用于回答“为什么学、将去哪里、现在在哪”。它不展开关内步骤、答案和完整知识目录，避免把未来任务一次性变成心理负担。课程可以根据学习记录调整某一关的练习，但不应频繁改动阶段成果和主线方向。

### 单一当前关

日常学习入口只展示当前一关，用一个可在短时间内完成的动作回答“现在做什么”。通常控制在 10–20 分钟，并具备：

- 一个清楚、可见的结果；
- 一个足够小的首次动作；
- 一次立即验证；
- 一条明确的通关证据；
- 中断后可以直接继续的位置。

把任务包装成小关卡，不是为了增加积分、排名或连续打卡压力，而是为了缩短“打开材料”到“得到反馈”的距离。一次只面对一个小目标、完成后再解锁下一关，可以降低启动阻力，也让进展由实际证据而不是阅读时长决定。

因此，推荐始终保留两个入口：

```text
大纲路线图：看方向、阶段、里程碑和进度
当前关卡：  看本次行动、验证方式和通关条件
```

学习者可以随时查看路线图，但日常不需要把全部未解锁任务展开成待办清单。

## 三种互补机制

### 渐进披露

第一次接触概念时，只解释当前任务需要的部分，同时明确哪些细节被延后。

简化不能变成错误描述。一个合格的初始解释应当：

- 当前足够使用；
- 后续深入时不需要推翻；
- 明确指出模型的边界，而不是暗示已经讲完。

例如第一次可以只说“handler 把内容写进响应”，等学习状态码时再展开响应结构，不必一开始介绍完整协议实现。

### 螺旋复现

课程围绕少量核心概念主线前进，而不是讲完一个主题后永久离开。概念每次复现时应满足至少一项：

- 换到新的真实场景；
- 增加一种操作；
- 增加一个边界或失败情况；
- 从“能看懂”提升到“能修改、排错或组合”。

如果练习内容和理解深度都没有变化，那只是机械重复，不属于螺旋学习。

### 提取练习

重复应优先要求学习者从记忆中取出知识，而不是重新阅读同一段说明。

可使用的动作包括：

- 不看答案说出上一次的流程；
- 在运行前预测结果；
- 找出负责某个行为的代码或步骤；
- 用自己的话解释结果；
- 在相似但不完全相同的场景中再次操作。

## 概念深度

同一个概念可以按下面的层次逐步推进：

| 层次 | 可观察表现 |
|---|---|
| 见过 | 知道它存在，能在材料中找到它 |
| 能追踪 | 能指出输入经过哪里、产生什么结果 |
| 能解释 | 能用自己的话说明作用和关键区别 |
| 能修改 | 能在已有结构中完成一个小改动并验证 |
| 能排错与组合 | 能定位常见问题，并与其他概念共同完成任务 |

并非所有概念都必须达到最高层。深度由最终目标决定，不为追求完整而增加负担。

## 微课设计

每个 10–20 分钟的学习单元只承担一个主要新层次，保持以下结构：

1. **结果**：先说明本次可以观察到的成果。
2. **提取**：回忆一个近期概念；合适时再回忆一个更早的概念。
3. **行动**：完成一个足够小的操作或修改。
4. **验证**：立刻获得可见、可比较的结果。
5. **加深**：只解释这次结果所需要的新一层。
6. **复述**：用自己的话解释发生了什么。
7. **记录**：写下已确认的理解、仍有的疑问和明确延后的内容。

可以用一句规则检查课程是否过重：

```text
一个旧锚点 + 一个新层次 + 一个明确延后点
```

“明确延后”很重要。它让学习者知道当前不理解某个细节是课程安排，而不是遗漏或失败。

## 重复节奏

重复不要求每日打卡，也不需要固定到精确天数。优先在有意义的使用场景中安排：

- **下一单元**：快速提取刚学过的核心流程。
- **间隔 2–3 个单元**：重新提取较早概念，并放进新的任务。
- **阶段结束**：只组合已经学过的内容，完成一次综合任务。
- **真实项目再次需要时**：从提示较多逐步过渡到独立使用。

连续两次都需要大量提示时，应缩小动作或插入同构练习；一次顺利完成则继续主线，不额外堆叠挑战。

## 课程设计流程

设计一条新学习路线时，按以下顺序进行：

1. 定义最终可观察成果，而不是先列知识目录。
2. 找出完成成果必须反复使用的 3–6 条概念主线。
3. 为每条主线确定需要达到的深度。
4. 用真实成果划分阶段和微课，让概念自然复现。
5. 为每一课标出旧锚点、新层次和延后点。
6. 根据学习记录调整下一课的练习，但不随意改变最终主线。

路线图负责回答“将去哪里”，概念主线负责回答“哪些能力正在逐层生长”，当前任务只回答“现在做什么”。三者不应混成一份庞大的待办。

## 教学者与学习者职责

### 教学者或 AI

- 提供样板、环境和安全的最小起点；
- 在解释前先让学习者观察或预测；
- 一次只引入一个主要新层次；
- 不因追求完整而提前讲解暂时用不到的细节；
- 根据实际表现记录深度，而不是根据“讲过了”判断掌握；
- 用学习记录调整练习，不轻易改动主线目标。

### 学习者

- 运行或完成实际操作；
- 观察结果与预期的差异；
- 至少亲自完成一个小改动；
- 尝试复述，不要求第一次就使用标准术语；
- 主动记录疑问，并区分“本次需要解决”和“可以延后”。

## 质量检查

设计或复盘一个学习单元时，检查：

- 是否有明确、可观察的结果？
- 是否同时提供简洁路线图和单一当前任务，并让两者职责分开？
- 当前关是否足够小，能快速开始、立即验证并在中断后继续？
- 是否只增加了一个主要新层次？
- 是否重新使用了至少一个旧概念？
- 重复是否带来了新场景、新操作或新深度？
- 是否要求学习者主动回忆、预测或解释？
- 是否明确写出暂时不需要深入的部分？
- 通关依据是实际表现，还是仅仅“看完了”？
- 本次疑问是否进入记录并影响后续练习？

## 常见反模式

- **没有全局方向**：只有临时设计的下一课，学习者不知道最终成果和当前所处位置。
- **路线图变成任务墙**：一次展开所有细节和待办，用未来工作量制造压力。
- **游戏化喧宾夺主**：用积分、排名或连续打卡代替真实理解和通关证据。
- **一次讲完**：第一次出现概念就解释全部语法、原理和边界。
- **知识目录驱动**：按术语列表前进，却没有最终可运行成果。
- **伪重复**：反复阅读或执行完全相同的步骤，没有新的判断。
- **只追新内容**：每一课都是新概念，旧知识从不重新调用。
- **过早独立**：刚见过一次就要求从空白开始完整实现。
- **提示永不减少**：每次都提供相同程度的逐步答案。
- **把讲过当掌握**：没有运行、修改、解释或排错证据。

