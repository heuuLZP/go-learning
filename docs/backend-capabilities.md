# 后端能力账本

> 这是教师用覆盖检查，不是学习者待办。只有真实场景出现时才解锁对应能力。

| 能力 | 状态 | 解锁场景 | 可观察成果 |
|---|---|---|---|
| HTTP 请求与响应 | 首月 | 当前关 | 能追踪请求进入 handler 并写回响应 |
| REST 与状态码 | 首月 | Todo 成为资源 | 能用正确方法和状态码完成 CRUD |
| SQL CRUD | 首月 | 内存数据需要持久化 | 能独立执行 `INSERT/SELECT/UPDATE/DELETE` |
| MySQL 持久化 | 首月 | SQL CRUD 已跑通 | 服务重启后仍能查询数据 |
| 事务 | 待解锁 | 一个动作需要同时写入两张表 | 能证明全部成功或全部回滚 |
| 索引 | 待解锁 | 出现稳定查询条件和足够数据 | 能用 `EXPLAIN` 对比索引前后的执行计划 |
| 认证与授权 | 待解锁 | Todo 引入用户和资源归属 | 能区分“你是谁”和“你能操作什么” |
| Redis | 待解锁 | 出现会话 TTL、缓存或限流需求 | 能解释键、过期时间和失效策略 |

## 固定取舍

- Go 是实现主线，通用后端概念是学习目标。
- Prisma 暂不学习；数据访问使用原生 SQL 和 `database/sql`。
- Redis 不作为“必须装上的组件”，只在它解决具体问题时加入。
- 新增后端技术前，必须先写清楚它替代了什么问题，以及不用它会怎样。
